SUMMARY: Patient Data Encryption and Audit Trail Test

DESCRIPTION:
This test case validates that patient data is encrypted using AES-256 and audit trails are stored for every database change.

OBJECTIVE:
Validate system compliance with FDA Part 11 and ISO 9001 requirements for encryption and audit trail storage.

PREREQUISITES:
1. System is configured to store patient records
2. Database is set up to track changes

TEST STEPS:
1. Step 1: Enter a new patient record
2. Step 2: Inspect the stored data to verify it is encrypted with AES-256
3. Step 3: Trigger a database change (e.g. update a patient's record)
4. Step 4: Observe system response to verify audit trail is created

EXPECTED RESULTS:
System behaves as per requirement, with patient data encrypted and audit trails stored for every database change.

TEST DATA:
Sample patient record data

ACCEPTANCE CRITERIA:
1. Data is encrypted with AES-256
2. Audit trail is created for every database change
